## 结论

已在原生 Anthropic 路径 `/v1/messages` 上完成兼容修复，目标是降低 Claude Code / Claude Desktop 通过本项目反代时的中断概率。

1. 移除重复 `message_stop` 的触发条件，只在上游未返回该事件时兜底补发。
2. `/v1/messages` 流式响应统一复用 `setSseHeaders()`，确保及时 flush SSE 头。
3. 新增 Anthropic-native 请求清洗：
   - 丢弃当前链路不兼容的 `output_config`
   - 规范化 `cache_control`，移除 `ephemeral.scope` 这类会触发校验失败的字段
4. 透传客户端的 `anthropic-version` / `anthropic-beta` 请求头，减少兼容性损失。
5. 非流式 `messages.create()` 增加 1 小时 timeout 兜底，规避大 `max_tokens` 时 SDK 直接拒绝。
